## Tips for Using These Resources Effectively

### Don't Try to Memorize Everything
- Programming is about problem-solving, not memorization
- It's okay to look things up constantly
- Even experienced programmers use references

### Practice is Key
- Reading alone won't make you a programmer
- Type code yourself, don't just copy
- Experiment and make mistakes
- Try to solve problems before looking at solutions

### Build Good Habits Early
- Use the cheat sheet until syntax becomes natural
- Learn to read error messages rather than just copy code
- Comment your code as you write it
- Test frequently rather than writing lots of code at once

### Ask Questions
- No question is too basic
- If you're confused, others probably are too
- Use these resources to help form specific questions
- "I got a NameError on line 5" is better than "It doesn't work"

## Quick Help Lookup

**Common Issues**:
- "My code won't run" ‚Üí Check for SyntaxError (missing colons, quotes)
- "Variable not found" ‚Üí Check for NameError (typo, created variable?)
- "Can't add string and number" ‚Üí Check for TypeError (convert with int/float)
- "Code not executing" ‚Üí Check for IndentationError (4 spaces after if)
- "Wrong answer in calculations" ‚Üí Check operator precedence (use parentheses)

**Quick Syntax Reminders**:
- Get number: `age = int(input("Age: "))`
- Print with variable: `print(f"You are {age} years old")`
- If statement: `if condition:` (don't forget colon!)
- Check if equal: `if x == 5:` (two equals signs!)
- Compare numbers: `<`, `>`, `<=`, `>=`, `==`, `!=`
